## What Is Fully Implemented and Verified

### 1. Data Pipeline — `src/data_loader.py`
- Loads the ChatDoctor dataset and performs a deterministic **70/30 split**
  (target corpus / shadow corpus) using `random_state=42`
- The split is done on **raw text strings before embedding**, which is the
  correct order — embeddings are generated downstream, not cached here
- Within the shadow set, a further 50/50 member/non-member split is applied
  with `random_state=11`
- Within the target set, a 50/50 member/non-member split is applied with
  `random_state=22`
- **No data leakage between splits has been verified**

### 2. Central DP Mechanism — `src/models/central_dp.py`
- Implements the **analytical Gaussian mechanism** for (ε, δ)-differential privacy
- Noise is calibrated exactly to the formula:
  `σ = 2√(2 ln(1.25/δ)) / ε`
- The factor of 2 in the numerator is the **global L2 sensitivity** of the
  embedding function, derived from the fact that all embeddings are
  L2-normalised to the unit hypersphere — the maximum possible L2 distance
  between any two points on a unit sphere is 2 (antipodal points)
- δ is fixed at `1e-5` across all experiments
- Noise is drawn freshly (not cached) on every call via `np.random.normal()`
- Noise is injected into the embedding matrix **before FAISS index construction**
- **Unit tested:** empirical noise variance matches theoretical σ² within 5%
  tolerance over 38,400 draws (100 samples × 384 dimensions), which is
  sufficient for CLT convergence

### 3. Local DP Mechanism — `src/models/local_dp.py`
- Implements a three-stage **Local Differential Privacy** pipeline to avoid the
  curse of dimensionality that would destroy utility if Gaussian noise were
  applied directly in 384-dimensional space
- **Stage 1:** Project from 384-d to 16-d via a **fixed random orthogonal
  linear map M₁**, initialised with `nn.init.orthogonal_` and frozen
  (`requires_grad=False`). M₁ is never trained and has zero dependence on
  private data
- **Stage 2:** L2-normalise the 16-d bottleneck vector, placing it on the
  15-dimensional unit sphere. This bounds the global L2 sensitivity in the
  bottleneck space to exactly Δ₂f = 2, the same bound as Central DP
- **Stage 3:** Inject calibrated Gaussian noise `N(0, σ²I₁₆)` independently
  per document using the same σ formula as Central DP. This is the formal
  (ε, δ)-LDP guarantee — it applies in the 16-d bottleneck space
- **Stage 4:** Decode back to 384-d via a second fixed random orthogonal map
  M₂, then L2-renormalise for FAISS compatibility
- The decoding (M₂) and final renormalisation are **data-independent functions
  of the noisy output** and therefore preserve the (ε, δ) guarantee without
  additional privacy loss, by the **Post-Processing Theorem** (Dwork & Roth
  2014, Proposition 2.1). This has been verified against the paper corpus
- **Unit tested:** output vectors lie on the unit sphere (norms ≈ 1.0 within
  1e-5 tolerance). Sensitivity bound tested with 10,000 random unit-vector
  pairs and 1,000 antipodal pairs — max observed L2 distance in bottleneck
  space is ≤ 2 + 1e-5 in all cases

### 4. Membership Inference Attack — `src/evaluation/mia_lira.py`
- Implements the **LiRA offline attack** from Carlini et al. (2022)
  "Membership Inference Attacks From First Principles"
- Uses **16 shadow models** (LogisticRegression), each trained on an
  independent bootstrap resample of the shadow member and non-member sets.
  Carlini et al. prove that 16 models with global variance estimation performs
  within ~20% of the full 64-model attack
- Each shadow model is a binary classifier trained on raw 384-d embeddings with
  label 1 = member, 0 = non-member
- After training, logit-transformed probabilities `φ(p) = log(p/(1-p))` from
  all 16 models on the full shadow non-member set are pooled into a flat array.
  `μ_out` and `σ_out` are fitted as the mean and standard deviation of this
  pooled array. This is the **global variance estimation** that Carlini et al.
  prescribe for the offline attack
- At evaluation time, probabilities from all 16 models are averaged per target
  point before the logit transform, producing a stable score estimate
- Privacy leakage is quantified as **TPR at 0.1% FPR**: the threshold τ is the
  0.1st percentile of non-member p-values, and TPR is the fraction of members
  whose p-value falls below τ. This metric is chosen over AUC because AUC
  averages across all operating thresholds including high-FPR regimes that are
  irrelevant to a real adversary (Carlini et al. 2022)
- **Important adaptation note:** Carlini et al. designed LiRA for classification
  models with softmax outputs. Our adaptation trains logistic regression shadow
  classifiers on raw embeddings and applies the logit transform to their output
  probabilities. This is a reasonable adaptation but has not been formally
  validated against a native embedding-space MIA baseline. The TPR values
  should be interpreted as approximate lower bounds on true worst-case leakage.
  This is documented in the manuscript Limitations section

### 5. Embedding Inversion Attack — `src/evaluation/inversion.py`
- Simulates a white-box embedding-to-text inversion attack using a secondary
  **clean FAISS reference index** built from unperturbed embeddings of the
  held-out reference corpus
- For each noisy embedding (post-DP), finds the nearest neighbour in the clean
  reference index and retrieves the corresponding text
- Reconstruction fidelity is measured via **ROUGE-L** between the source text
  and the retrieved nearest-neighbour text
- This is a nearest-neighbour approximation of a full inversion attack (e.g.,
  Vec2Text by Morris et al. 2023). It underestimates the true inversion risk
  but is computationally feasible within the project scope. Documented in
  Limitations

### 6. Utility Metric — `src/evaluation/utility.py`
- Measures end-to-end semantic preservation using **BERTScore** with the
  `roberta-large` backbone (pretrained, not fine-tuned)
- BERTScore is computed between the original source text and the nearest-
  neighbour text retrieved from the noisy index
- Precision, Recall, and F1 are all computed and written to results.csv
- BERTScore is used as the primary utility metric over exact-match accuracy
  because conversational medical QA has many valid paraphrases and exact-match
  is brittle

### 7. Experiment Orchestration — `scripts/run_experiments.py`
- Global random seeds set at entry: `np.random.seed(0)`, `torch.manual_seed(0)`
  for full reproducibility across noise draws (not just data splits)
- Runs a **baseline pass** (unperturbed embeddings, ε = ∞, mechanism =
  "Baseline") before the ε sweep, providing the upper anchor for tradeoff curves
- ε sweep: `[0.1, 1.0, 5.0, 10.0]` — four operating points
- For each ε, runs both Central DP and Local DP and evaluates all three metrics
  (MIA TPR, inversion ROUGE-L, BERTScore F1)
- Writes results in **long format** to `data/results.csv` with schema:
  `epsilon | mechanism | tpr_mia | inversion_rouge_l_mean | bert_precision | bert_recall | bert_f1`
- `LiRAMembershipInference` instantiated with `n_shadow_models=16` explicitly

### 8. Plotting Engine — `scripts/plot_results.py`
- Reads the long-format `results.csv` and generates three separate
  high-resolution figures:
  - `utility_vs_epsilon.png` — BERTScore F1 vs ε (Central DP vs Local DP)
  - `mia_vs_epsilon.png` — TPR@0.1%FPR vs ε (Central DP vs Local DP)
  - `inversion_vs_epsilon.png` — ROUGE-L vs ε (Central DP vs Local DP)
- x-axis is on a **log scale** (appropriate for ε ∈ [0.1, 10])
- Central DP and Local DP are overlaid on the same axes with `hue="mechanism"`
- Baseline value is drawn as a **horizontal dotted line** on each utility plot,
  consistent with the convention in Bollegala et al. (2025)
- Output written to `docs/figures/`

## Key Technical Decisions Made (and Why)

| Decision | What we chose | Why |
|---|---|---|
| LDP dimensionality | 16-d bottleneck | Du et al. (2023) empirically show 16-d gives best utility; curse of dimensionality makes full 384-d LDP unusable |
| M₁, M₂ initialization | Fixed orthogonal (`nn.init.orthogonal_`) | JL-style projection with known geometric properties; defensible in manuscript; zero data dependence |
| MIA metric | TPR @ 0.1% FPR (not AUC) | Carlini et al. (2022): AUC hides localised privacy breaches; low-FPR TPR is the correct worst-case bound |
| Number of shadow models | 16 | Carlini et al. offline variant: 16 models with global variance estimation is within 20% of full 64-model attack |
| δ value | 1e-5 | Standard in DP literature for datasets of this scale |
| ε sweep | [0.1, 1.0, 5.0, 10.0] | Covers strong privacy to near-baseline range; log-spaced for interpretable curves |
| Utility metric | BERTScore (roberta-large) | Exact match is brittle for conversational medical QA; BERTScore captures semantic similarity |
| Inversion metric | Nearest-neighbour ROUGE-L | Vec2Text is computationally prohibitive; NN approximation is feasible and directionally correct |
| Baseline representation | ε = ∞ in results.csv | Convention from Carlini et al. (2022) and Du et al. (2023) |
